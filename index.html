Astroとは

AstroはSSGの1つで2022年の8月にver1.0 がリリースされてから現在だとすでにver2.1まで出ている。 既にGitHubのスターは約29kくらいある。※ちゃんとスター送っといた◎

特徴

  • デフォルトでjsをHTMLに変換していく
    Astroはビルド時にjsをHTMLに変換して行くので、処理が早くなる。
    Partial Hydrationをその都度実施しないと動かない（Storeの処理が会ったりVueやReactで構築されたコンポーネントを使用するときなど）
  • 独自のアイランドアーキテクチャを採用している
    ページ中の「静的な部分」や「動的な部分」をそれぞれアイランドという独立なコンポーネントに分割し、コンポーネント指向な開発ができる。\
  • インテグレーションを追加すると簡単にいろいろな機能の追加ができる
    VueやReactなどで作成されたフレームワークのコンポーネントを追加したりTailwindなどのライブラリの追加が可能。
  • MPAで作成されSPAよりも早い
    NuxtのようにPagesディレクトリ配下に.astroファイルや.md、.mdxファイルを配置することでビルド時にpathを設定してくれる。

package.jsonを修正する

個人的にpackage.jsonがいけてない感あったのでscript周りを修正する。

{
    "scripts": {
        "start": "astro dev",//"dev": "astro dev"から修正
        "build": "astro check && tsc --noEmit && astro build",//"build": "astro build"から修正
        "preview": "astro preview",
    }
}

package scriptの説明

  • start
    ローカル実行用コマンド
  • build
    ビルド用コマンド
    astroのコードにエラーがないか確認し、buildファイルを作成する。（デフォルトでdistファイルに作成される。）
    今回はgithub pagesに公開するので、下記のようにsiteの設定とbase pathの設定を追記する 
        export default defineConfig({
        integrations: [vue({
            appEntrypoint: '/src/pages/_app'
        }), vuetify(), react(), mdx(), markdoc()],
        site: 'https://haishikawa.github.io/',//ここを追加
        base: '/Astro-Demo'//ここを追加
    });
  • preview
    ビルドファイル確認コマンド
    ビルドコマンドで作成したファイルをローカル実行で確認することができる。

下記コマンドで実行して確認する

Vueを使う

Vueのコンポーネントを使ってみる。

以前、案件で作成した下記のVueコンポーネントを使ってみる

//StatusChip.vue
<template>
    <v-chip
        class="status-chip"
        :color="getStatusColor(statusCode)"
        :outlined="isOutlinedStatus(statusCode)"
        :text-color="getStatusFontColor(statusCode)"
    >
        <slot></slot>
    </v-chip>
</template>

<script lang="ts">
import { Component, Prop, Vue } from 'vue-property-decorator';
import { Status } from '@/enum';

@Component
export default class StatusChip extends Vue {
    @Prop()
    private statusCode!: number;
    getStatusColor(status_code: number) {
    switch (status_code) {
        case Status.unApplicated:
        case Status.applicated:
        return 'default';
        case Status.approved:
        return 'positive';
        case Status.repudiated:
        return 'negative';
        case Status.systemError:
        return 'warning';
    }
    }
    isOutlinedStatus(status_code: number) {
    switch (status_code) {
        case Status.unApplicated:
        case Status.applicated:
        return true;
        default:
        return false;
    }
    }
    getStatusFontColor(status_code: number) {
    switch (status_code) {
        case Status.unApplicated:
        case Status.applicated:
        case Status.systemError:
        return 'default';
        default:
        return 'white';
    }
    }
}
</script>

astro.config.mjsにVueとVuetifyのインテグレーションの設定追加

import { defineConfig } from 'astro/config';
import react from '@astrojs/react';
import vue from '@astrojs/vue';
import vuetifyPlugin from 'vite-plugin-vuetify';
import mdx from "@astrojs/mdx";
import markdoc from "@astrojs/markdoc";

/**
* Vuetify integration for Astro
* @param {import('astro/config').Options} options
* @returns {import('astro/config').AstroIntegration}
*/
function vuetify(options) {
return {
    name: 'my-astro-vuetify-integration',
    hooks: {
    'astro:config:setup': ({
        updateConfig
    }) => {
        updateConfig({
        vite: {
            ssr: {
            noExternal: ['vuetify']
            },
            plugins: [vuetifyPlugin()]
        }
        });
    }
    }
};
}

Store使う

Vueが使えることはわかったし公式のtutorialにも記載あるのでまあ、いろいろフレームワーク入れれるのはわかったが、これStateの管理はどうするんだ？
VueだとVuexとかReactだとReduxとかそれぞれ別々の状態管理していたと思う。ので気になったので調べて使ってみた。

nanostores

どうやら公式はnanostoresを使用して状態の管理を実施してほしいらしい 以下のコマンドでインストール

npm i nanostores
npm install nanostores @nanostores/persistent //localStorageに値を保持するためのライブラリ

storeのファイルを作成してmutationとかの処理を追加

//stores/counter.ts
import { persistentAtom } from '@nanostores/persistent';

const initialValue = { value: 0 };
//MPAなのでページ間でStoreの値を参照、更新できるよう設定
const counter = persistentAtom<number>('counter', initialValue.value, {
    encode: JSON.stringify,
    decode: JSON.parse,
});

const increaseCounter = () => counter.set(counter.get() + 1);

const decreaseCounter = () => counter.set(counter.get() - 1);
const cleanCounter = () => counter.set(initialValue.value);
export { counter, increaseCounter, decreaseCounter, cleanCounter };

reactで使う

reactで使う場合は下記のコマンドを実行して追加でライブラリを入れる

npm i nanostores @nanostores/react

※reactのインテグレーションの設定をしていない場合は下記のコマンドを実行して設定する

npx astro add react

コンポーネントで使用する

import { useStore } from '@nanostores/react';
import { counter, increaseCounter, decreaseCounter } from '../stores/counter';

export default function React() {
    const count = useStore(counter);
    const name = 'React';
    return (
        <>
            <p> {name} のコンポーネント</p>
            <button onClick={decreaseCounter}>-</button>
            <pre>{count}</pre>
            <button onClick={increaseCounter}>+</button>
        </>
    );
}

vueで使う

layoutsに追加する

公式は下記のようにpagesに認証している場合のみ遷移させるlayoutを実装することを推奨していた

<script>
    let isLoggedIn = localStorage.getItem('isLoggedIn');//認証情報を保持しているか確認する

    if (isLoggedIn === null || isLoggedIn === '' || isLoggedIn === 'false') {
        window.location.href = import.meta.env.BASE_URL + 'login';
    }
</script>
